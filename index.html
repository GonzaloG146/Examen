Trabajo Académico

Metodologías de Desarrollo
de Software

Un recorrido por los enfoques más utilizados para planificar, desarrollar y mantener aplicaciones informáticas de manera eficiente.

1

¿Qué es una Metodología?

Es un conjunto de procesos, técnicas y herramientas que ayudan a planificar, desarrollar y mantener aplicaciones informáticas.

Ejemplo

Una empresa que crea una app para gestionar citas médicas usa una metodología para organizar tareas, distribuir responsabilidades y controlar el avance del proyecto.

2

Marcos de Trabajo Tradicionales

Modelo Cascada

Metodología secuencial donde cada fase debe completarse antes de iniciar la siguiente.

Análisis de requisitos Diseño Implementación Pruebas Mantenimiento
Ejemplo

Una universidad solicita un sistema de matrículas. Se recopilan todos los requisitos, se diseña el sistema, se programa y se realizan pruebas antes de ponerlo en funcionamiento.

RUP — Rational Unified Process

Metodología iterativa que divide el proyecto en fases para mejorar el control y reducir riesgos.

Inicio Elaboración Construcción Transición
Ejemplo

Una aplicación bancaria se construye por etapas: primero el módulo de usuarios, luego el de consultas y posteriormente las transferencias.

3

Marcos de Trabajo Ágiles

Manifiesto Ágil

Busca entregar software funcional rápidamente y adaptarse con facilidad a los cambios del cliente.

Individuos e interacciones Software funcionando Colaboración con el cliente Respuesta al cambio
Ejemplo

Un cliente solicita cambios durante el desarrollo de una app móvil. El equipo adapta el proyecto sin esperar a finalizar todo el sistema.

Programación Extrema — XP

Metodología ágil enfocada en la calidad del software y la mejora continua a través de valores claros.

Comunicación Simplicidad Retroalimentación Coraje Respeto
Ejemplo

Dos desarrolladores programan juntos un sistema de ventas: uno escribe código y el otro revisa posibles errores y mejoras en tiempo real.

Scrum

Divide el proyecto en ciclos cortos llamados Sprints para entregar funcionalidades de forma continua e incremental.

Product Owner Scrum Master Equipo de Desarrollo Sprint Daily Meeting
Ejemplo

Un equipo construye una tienda virtual: Sprint 1 crea el inicio de sesión, Sprint 2 desarrolla el catálogo y Sprint 3 implementa el carrito de compras.

4

Planeación de Proyectos de Software

La planificación permite estimar recursos, tiempos, costos y riesgos antes de iniciar el desarrollo.

Ejemplo

Para una app de pedidos a domicilio, la empresa calcula que necesitará cuatro meses, tres desarrolladores y un diseñador para completar el proyecto.

★

Caso Práctico — Clínica de Citas Médicas

Una clínica desea crear una aplicación para reservar citas médicas. Así aborda el problema cada metodología:

Cascada Se definen todos los requisitos antes de comenzar a programar.
RUP El sistema se construye mediante iteraciones controladas.
XP Pruebas constantes y programación en pareja durante todo el proceso.
Scrum El trabajo se organiza en Sprints de dos semanas con entregas continuas.

Cada metodología ofrece ventajas distintas según las necesidades del proyecto y el nivel de flexibilidad requerido.
